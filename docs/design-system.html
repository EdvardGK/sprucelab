Rules & Decisions
Do
  • Use CSS variable tokens for all status colors
  • Use 8px grid for spacing (multiples of 4px)
  • Use clamp() for dashboard text/icons
  • Use radius scale: --radius-sm (6px) buttons, --radius-md (8px) inputs, --radius (12px) cards
  • Glassmorphism for sidebar and floating panels only
  • Cards are opaque white with subtle border
  • Top accent bar on every main content area
  • Use traffic light tokens, not raw Tailwind colors
Don't
  • Don't use bg-emerald-50 / text-amber-600 for status
  • Don't apply glassmorphism to regular content cards
  • Don't use Tremor components in new code
  • Don't hardcode Norwegian strings (use i18n)
  • Don't use shadows heavier than the card hover shadow
  • Don't mix font sizes outside the 6-role hierarchy
  • Don't hardcode border-radius values — use --radius-sm/md/radius
Transition defaults
Color / border0.15s ease
Transform / shadow0.2s ease
