1 · Triangle gradient (lime → forest → navy)

Polyline through OKLCH color space, three vertices. Doubles the perceptual range of a single-segment gradient. The hue rotation at the forest vertex (155° → 260°) acts as the discipline cliff — classes on either side don't perceptually blur together.

6 · Subdivision pattern · van der Corput / bit-reversal (base 2)

Treemaps and multi-item charts don't need canonical positions; they need predictable subdivision of the gradient so adjacent classes never collide. Bit-reversal sequence gives maximal-separation samples for any N — each new sample falls in the biggest empty interval.

The sequence is purely deterministic — slot N always gets the same t. Adding/removing classes never reshuffles existing assignments; new ones fall into the gaps. Compare to even subdivision t = i/(N-1) which reshuffles every position when N changes.

7 · Generator vs assignment · two-stage model

The gradient curve and subdivision pattern are fixed mathematics. Which class gets which slot is a separate decision — canonical table for the main UI, stable hash for the long tail.

Stage 1 · Generator (fixed)
Triangle gradient + bit-reversal sequence. Same code emits the same set of 32 colors every time. 
function paletteSlot(i) {
  const t = bitReverse(i);     // deterministic
  return sampleTriangle(t);    // OKLCH sample
}

// SLOTS = [paletteSlot(0), paletteSlot(1), ...]
Stage 2 · Assignment (per surface)
Main UI uses the canonical table from §2. Treemaps and multi-item charts use a stable hash on the class name. 
// Canonical (main UI, chips, KPIs)
classColor(c) = SLOTS[CANONICAL_IDX[c]]
              ?? SLOTS[hash(c) % 32]

// Chart-only (treemap, sparkline stack)
chartColor(c) = SLOTS[hash(c) % 32]

Why this works for treemaps: the assignment is "random" relative to discipline (a wall and a duct can land on adjacent slots) but the colors themselves are well-distributed across the gradient. No two slots are within ΔE_min of each other until N exceeds the gradient's capacity. Below that ceiling, every slice of the treemap reads as a distinct shade.

9 · Status colors · traffic lights, callouts, contrast budget

Status colors live outside the data gradient's bounding box so they can never be confused with a class swatch. Five families, each at three intensities (solid · tinted · text). All five share the same L target band (L=0.58–0.78 for solid, L=0.94–0.96 for tinted) so the system reads as a single tonal family.

Best-practice notes (cross-referenced)
  • WCAG 2.2 4.5:1 minimum for status text on tinted backgrounds (normal body text); 3:1 for ≥18pt or icon-only. All five families clear 4.5:1 against their tinted-background pair by design (L=0.40 fg on L=0.94–0.96 bg).
  • NN/g Never color alone — every status pairs with a glyph (✓ ! ✕ ℹ ◆ —) so red/green colorblind users (~5% of male users) still read the same message.
  • Material 3 Tonal pairing — solid (L≈0.58–0.78) + tinted background (L≈0.94–0.96) + text (L≈0.40). Same hue family at three L stops produces the callout's "container" pattern without bespoke art per status.
  • Apple HIG Reserve red for destructive / hard-fail only. Don't use it for "incomplete" or "warning" — that's amber's job. We follow this: danger = verification hard-fail, warning = missing classification / soft gap, neutral = not yet analysed.
  • Tableau Sequential vs categorical separation — the data gradient (lime → forest → navy) is sequential; the status palette is categorical. They live in different sub-spaces of OKLCH so the eye reads them differently. Gradient cells in a treemap never "look like" a status pill.
  • Hue spacing danger 25° · signal 50° · warning 85° · success 145° · info 240°. Smallest gap (signal ↔ warning) is 35°; differentiated additionally by chroma (signal saturated 0.18 / warning 0.16) and L (both ~0.78 — distinguished by hue alone here, OK because the glyph carries the meaning).
  • Traffic-light Threshold ladder for percent KPIs: green ≥ green_threshold, amber ≥ amber_threshold, red below. Per-KPI thresholds live in ProjectConfig — the color is a render decision, not a meaning decision.
