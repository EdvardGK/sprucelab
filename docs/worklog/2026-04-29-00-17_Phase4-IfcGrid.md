## Technical Details

### IfcLine `Dir` is an `IfcVector`, not an `IfcDirection`
First test run failed: `axis['direction']` came back `None` even though
the fixture authored a real direction. `IfcLine.Dir` is an `IfcVector`,
which has `Orientation` (the `IfcDirection`) and `Magnitude` (a float in
the model's length unit) — `DirectionRatios` lives on `Orientation`,
not on the `IfcVector` itself. The parser now branches on
`direction.is_a('IfcVector')` and falls through to a raw `IfcDirection`
for tolerance against unusual authors. Magnitude is multiplied by
`length_unit_scale` so the resulting direction vector is in meters,
matching the `start` point.

### Placement scaling
`ifcopenshell.util.placement.get_local_placement` returns a numpy 4×4
in the model's authored units. We copy to a nested `list[list[float]]`
(JSON-friendly) and scale only the translation column (`m[0..2][3]`) to
meters. Rotation/scale columns stay unitless. This matches how storey
elevations are converted on line 307 of the same file.

### Failure isolation
Grid extraction is wrapped in `try/except` in `parse_types_only`. A bad
grid shouldn't fail the whole types extraction — types-only parsing is
the load-bearing path that downstream features depend on. On failure we
write `discovered_grid = {'grids': []}` and log a `warning` entry, same
shape the unit-extraction code uses for its own failure case.

### Test sys.path tweak
The ifc-service uses bare imports (`from services.ifc_parser import ...`,
`from repositories.ifc_repository import ...`) because it runs under
uvicorn from its own cwd. Pytest runs from the repo root, so importing
`services.ifc_parser` only works if `backend/ifc-service/` is on
`sys.path`. The conftest already had `BACKEND` (for Django) but not
`IFC_SERVICE`; one extra `sys.path.insert` keeps the parser-level unit
tests honest without forcing them to do path gymnastics.
