### Findings / surprises

1. **`/api/types/types/` returns a bare list, not a paginated DRF envelope.** This is not a bug — the CLI's `types list` code handles both (`rows = body.get('results', body) if isinstance(body, dict) else body`). The `--json` output is the raw API response.
2. **Model UUIDs in `types list` are from the `models` app** (`Model` table), not `SourceFile` UUIDs. The CLI flag `--model` correctly maps to the right field.
3. **`spruce types list` needs the model ID from `GET /api/models/`**, not from `GET /api/files/`. These are different objects.
4. **DEV_AUTH_BYPASS=1** required for smoke (no token in dev keyring). The CLI's `_auth.py` falls back to None gracefully and the backend's DevBypassAuthentication handles the unauthenticated request.
